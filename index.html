Prototype with Agility

I'm almost sure that you have lived the situation where front-end developers are waiting the startup of a external service to begin some tests.

If you lived it, you've been always thinking in a way to elaborate a mock of backend services, actually. In order to help you in these situations, vaporwave comes to the ground.

Github

Why use vaporwave?

Develop integrated interfaces

You can advance integrating your front-end application with vaporwave mock server.

Simplicity

For very simple applications, you don't need a google account to startup a firebase backend, you just need vaporwave

Evolve

The tool is been projected to evolve in a continuous flow. Currently, it has been design a way of use vaporwave as a serious tool for the backend. Check the github and contribute for its evolution.
